# knights

Visualizes chess-knight problems on a number-spiral grid (from a Numberphile video ‚Äî
see `KNIGHTS.md` for the canonical problem statements and validated outcomes), plus a
general, config-driven placement engine that the multi-color games are presets of. A
Cargo **workspace**: `core` (the engine, a library) + `cli` (headless/batch renders);
a `web` front-end (egui ‚Üí WASM, interactive editor + display) is in progress. Output is
hand-rolled indexed PNG; TOML configs use `serde` + `toml`. (No SVG ‚Äî it was removed.)

> Keep this file thin and pointer-shaped. The *how* and *why* live in each module's
> doc-comment; the facts that must stay true live in the tests. Don't restate either
> here ‚Äî a standalone doc is the first thing to drift.

## Layout

`core/src/` (library `knights_core`, the engine ‚Äî pure compute + PNG bytes, WASM-ready):
- `spiral.rs` ‚Äî square-spiral coordinates: `SpiralWalker` (O(1) stepper, any of the 8 orientations via `oriented`) and `Spiral` (cached tables).
- `piece.rs` ‚Äî piece types (finite attack-offset lists) and the cell-byte `KindTable` ((type, color) ‚Üí one `u8`).
- `engine.rs` ‚Äî the general placement engine: round-robin pieces on their own spirals, the backward attack check, `PlacementResult`, and the `Board` render trait.
- `config.rs` ‚Äî TOML config schema (serde) ‚Üí validated `EngineConfig`.
- `redblack.rs` ‚Äî Problem 3 + variants, thin presets (`Variant::engine_config`) over `engine`.
- `knight.rs` ‚Äî Problem 1, the trapped knight (and the 8 knight offsets, shared).
- `courteous.rs` ‚Äî Problem 2, courteous knights + cluster grouping.
- `raster.rs` ‚Äî streaming indexed-color PNG (any `Board`) + the courteous rasterizer.

`cli/src/main.rs` ‚Äî CLI front-end over `knights_core`: subcommand dispatch + flag parsing, PNG output (and trapped's text result).
`web/` ‚Äî egui‚ÜíWASM front-end (`eframe`); currently a viewer (radius/variant + Simulate). Run with `trunk serve` ‚Äî see `web/README.md`. Excluded from the default workspace build (heavy deps); built via `trunk` or `cargo ‚Ä¶ -p knights-web`.
`examples/*.toml` ‚Äî sample `custom` configs (canonical redblack, four-color, mixed piece types).

Each module's doc-comment explains its design ‚Äî start there.

## Run / test

- `cargo test` ‚Äî whole workspace; includes the verification oracles below.
- `cargo run -- [trapped|courteous|redblack|custom] [flags]` (default: `trapped`). `trapped`
  prints text only; the board problems write a PNG into `out/`.
- `cargo run --release -- redblack --radius 2000` for large boards.
- `cargo run -- custom --config examples/mixed-pieces.toml` ‚Äî arbitrary games; the TOML
  schema (piece types + pieces with offsets/spiral/color) lives in `core/src/config.rs`.

Flags: `--radius <r>`, `--start <n>` (trapped only), `--squaresize <px>` (PNG pixels/square,
default 1), `--variant canonical|rot180|mirror|quad` (redblack only; `rot180`/`mirror` reorient
Red's spiral, `quad` plays four colors ‚Äî all non-canonical experiments, see `redblack.rs`),
`--config <path>` (required by `custom`), `--out <path>`. See `cargo run -- --help`.

## Verification oracles (enforced by tests ‚Äî trust these, not prose)

- Trapped knight traps at square **2084** (center = 1; OEIS A316667) ‚Äî `knight::tests::trapped_at_2084_with_start_1`.
- Courteous = OEIS **A308885** ‚Äî `courteous::tests::matches_oeis_a308885`.
- Red & Black = OEIS **A392177** (Black) / **A392178** (Red) ‚Äî `redblack::tests::matches_oeis_a392177_a392178`.

## Conventions

- A board cell is a `u8` ‚Äî a `KindTable` index that is also the palette index; the PNG
  encoder picks the bit depth from the palette size, so more piece types need no encoder
  change. A piece's attack offsets need not be symmetric; "attacked" is read from the
  *attacker's* offsets (`engine::next_spot`).
- Dependencies are fine (the earlier "std-only" rule came from another project). Current
  deps: `serde` + `toml`, for config parsing.
- `out/` and `target/` are gitignored. `out/` may hold images generated by hand ‚Äî check
  a file's contents before overwriting it.
